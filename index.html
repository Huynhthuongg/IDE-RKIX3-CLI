AndroidX Contribution Guide

Một trang docs tiếng Việt cô đọng cho quy trình lấy source, mở Android Studio, build, kiểm thử và upload thay đổi AndroidX — trình bày theo phong cách sạch, tập trung và có code card giống agents.md.

Bắt đầu nhanh Tài liệu gốc 
# AndroidX quickstart
mkdir androidx-main
cd androidx-main

repo init -u https://android.googlesource.com/platform/manifest \
  -b androidx-main \
  --partial-clone \
  --clone-filter=blob:limit=10M

repo sync -j8 -c
cd frameworks/support
ANDROIDX_PROJECTS=MAIN ./gradlew studio
MAIN Project set khuyến nghị cho phần lớn thư viện Jetpack.
repo sync Lệnh đồng bộ mã nguồn tương tự tinh thần của git fetch.
createArchive Tạo Maven artifacts cục bộ để thử trong ứng dụng riêng.

Vì sao cần trang này?

README dài được chuyển thành luồng thao tác dễ quét.

Tài liệu AndroidX có nhiều bước rời rạc: cấu hình Git, tạo checkout, đồng bộ bằng repo, mở đúng Android Studio, build toàn bộ thư viện, chạy test và gửi thay đổi lên Gerrit. Trang này gom các bước đó thành các khối độc lập để lập trình viên hoặc AI coding agent có thể làm theo nhanh hơn.

Quickstart

4 bước đầu tiên

01

Cấu hình Git

Đặt tên và email để commit trong checkout AndroidX.

git config --global user.name "Your Name"
git config --global user.email "you@example.com"
02

Tạo checkout

Khởi tạo repo AndroidX bằng branch androidx-main và clone filter.

mkdir androidx-main
cd androidx-main
repo init -u https://android.googlesource.com/platform/manifest \
  -b androidx-main --partial-clone \
  --clone-filter=blob:limit=10M
03

Đồng bộ mã nguồn

Tải phần mã cần thiết cho quá trình build và phát triển AndroidX.

repo sync -j8 -c
04

Mở Android Studio

Dùng wrapper Gradle để mở phiên bản Studio được khuyến nghị.

cd path/to/checkout/frameworks/support
ANDROIDX_PROJECTS=MAIN ./gradlew studio

Build & kiểm thử

Tạo artifacts, chạy test và kiểm tra sample app.

Build hoàn chỉnh

Chạy Gradle từ thư mục frameworks/support để build toàn bộ thư viện.

./gradlew createArchive

Dùng Maven local trong app của bạn

Thêm repository build ra vào đầu danh sách repositories của project app.

maven { url '/path/to/checkout/out/androidx/build/support_repo/' }

Chạy một test cụ thể

Mở file test trong Android Studio, nhấp chuột phải vào class hoặc method @Test, rồi chọn Run.

Chạy sample app

Các app thường có hậu tố -integration-tests-testapp hoặc dạng support-*-demos.

Đóng góp

Từ nhánh làm việc đến review trên r.android.com

1

Chuẩn bị tài khoản

Tạo mật khẩu HTTPS và chấp thuận Google Contributor License Agreement trước lần upload đầu tiên.

Tạo mật khẩu HTTPS
2

Tạo nhánh và commit

repo start my_branch_name .
# chỉnh sửa cần thiết
git commit -a
3

Upload để review

repo upload --current-branch .
4

Thêm reviewer

Mở r.android.com, đăng nhập và chọn reviewer phù hợp dựa trên lịch sử git hoặc file OWNERS.

FAQ

Các lưu ý hay gặp

Có những project set nào?

Có thể dùng ALL, MAIN, COMPOSE hoặc FLAN qua biến ANDROIDX_PROJECTS.

Khi Studio báo “Unregistered VCS root detected” thì sao?

Chọn “Add root” để bật tích hợp Git cho Android Studio trong checkout AndroidX.

Nếu thấy underline đỏ trong IDE?

Chạy Build > Clean Project trước khi tiếp tục điều tra lỗi thật.

Binary dependencies nằm ở đâu?

AndroidX lưu bản sao Gradle binary libraries trong prebuilts/androidx/internal và prebuilts/androidx/external.

Thiết kế tham khảo bố cục tối giản, code-first của agents.md.
